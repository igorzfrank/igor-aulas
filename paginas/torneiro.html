3 📋 CÓDIGO BASE — Copie o HTML e o CSS
🧠 Como usar os códigos abaixo
Crie uma pasta no seu computador. Dentro dela, crie dois arquivos: index.html e style.css. Copie o HTML para o arquivo .html e o CSS para o arquivo .css. Depois é só abrir o HTML no navegador e começar a modificar o CSS com a identidade da sua empresa.
HTML index.html 📋 Copie este código
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <link rel="stylesheet" href="style.css" />
  </head>
  <body>
    <header>
      <h2>LOGO</h2>
      <div class="links">
        <a href="#">Home</a>
        <a href="#">Sobre</a>
        <a href="#">Contato</a>
        <a href="#">Ajuda</a>
      </div>
    </header>
    <div class="hero">
      <h1>HERO SECTION</h1>
    </div>
    <div class="cards">
      <!-- Card 1 -->
      <div class="card">
        <div class="card-img"></div>
        <p>Lorem ipsum dolor sit amet...</p>
        <button>Botão</button>
      </div>
      <!-- Repetir para cards 2 a 9 -->
    </div>
    <footer>
      <p>© 2026 Pagina Marketplace - Projeto Educacional HTML e CSS</p>
    </footer>
  </body>
</html>

4 📏 REGRAS, GITHUB, CANVA E ENTREGA
✅ O que vocês DEVEM modificar
🎨 Cores — Backgrounds, textos, botões, header e footer
🔤 Tipografia — Família de fontes, tamanhos, pesos
📏 Espaçamentos — Paddings, gaps, margens
🖼️ Card-img — Substituir a cor cinza por imagens ou cores da marca
🎯 Botões — Cores, bordas, efeitos de hover
📐 Hero section — Background, cor do texto, altura
❌ O que NÃO podem alterar
🚫 Estrutura HTML — Não adicione nem remova tags
🚫 Ordem dos elementos — Mantenha header, hero, cards e footer
🚫 Classes e IDs — Não renomeie os seletores existentes
🚫 Texto do conteúdo — Mantenha os textos como estão
🚫 Layout geral — Flexbox e grid já definidos devem ser mantidos
🔗 GitHub — Criando o Repositório da Equipe
A equipe deve criar um novo repositório no GitHub para armazenar o código do projeto. Siga os passos abaixo:
1. Um integrante da equipe cria um novo repositório no GitHub (público).
2. Nomeie o repositório como: landing-page-[nome-da-empresa] (ex: landing-page-nubank).
3. Adicione os outros integrantes como colaboradores do repositório.
4. Façam o upload dos arquivos index.html e style.css.
5. Ao final, enviem o link do repositório junto com a entrega.
🎤 Canva — Apresentação do Projeto
A equipe deve criar uma apresentação no Canva contendo os recursos que foram aplicados no projeto. A apresentação deve incluir:
1. Slide 1 — Capa: Nome da empresa, nome dos integrantes da equipe.
2. Slide 2 — Referências: Logo da empresa, paleta de cores utilizada, fontes escolhidas.
3. Slide 3 — Antes e Depois: Comparação do wireframe neutro vs wireframe personalizado.
4. Slide 4 — Destaques: Principais mudanças feitas no CSS (cores, fontes, botões, cards).
5. Slide 5 — Resultado Final: Print da página finalizada + link do repositório GitHub.
✅ Como fazer
1. Acessem os links do Figma para pesquisar a identidade visual da empresa.
2. Copiem o HTML e CSS base para uma pasta do projeto.
3. Criem o repositório no GitHub e adicionem os colaboradores.
4. Modifiquem o CSS aplicando cores, fontes e estilo da marca.
5. Montem a apresentação no Canva com os recursos aplicados.
📄 O que entregar
🔗 Link do GitHub — Repositório com os arquivos HTML e CSS.

🎤 Link do Canva — Apresentação com os recursos aplicados.

📄 Formato — Cole os links em um PDF ou envie diretamente.

📤 Envio online — Envie pelo ambiente virtual da disciplina.
📊 Critérios de Avaliação
✅ Fidelidade à marca — As cores e estilo refletem a identidade da empresa?
✅ Consistência visual — As cores e fontes foram aplicadas em toda a página?
✅ HTML preservado — A estrutura HTML não foi modificada?
✅ GitHub — O repositório foi criado e o link está acessível?
✅ Apresentação Canva — A apresentação contém todos os slides solicitados?
💡 Dica final: Dividam as tarefas! Enquanto um integrante cria o repositório no GitHub, outro pesquisa as cores e fontes da empresa, e outro já começa a modificar o CSS. A apresentação no Canva pode ser feita em paralelo. Trabalho em equipe é a chave! 🚀
